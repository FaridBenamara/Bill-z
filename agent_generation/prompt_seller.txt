Génère 5 factures différentes, chacune correspondant à une vente effectuée par Altevia Solutions.
Chaque facture doit respecter strictement la structure définie dans le contexte.

Le résultat final doit être un unique JSON de la forme :

{
  "invoice_1": { ... },
  "invoice_2": { ... },
  ...
  "invoice_5": { ... }
}


Consignes supplémentaires :

Pour chaque facture, invente un client crédible (buyer).
Chaque client doit avoir un SIRET et une TVA réalistes à renseigner dans les champs client_siret et client_vat.

Seller doit TOUJOURS être EXACTEMENT (JAMAIS varier) :

"name": "Altevia Solutions" (TOUJOURS exactement ce nom, jamais varier)

"address": "12 Rue des Entrepreneurs, 75015 Paris, France" (TOUJOURS cette adresse exacte, JAMAIS varier, JAMAIS inventer d'autre adresse)

"email": "contact@altevia.com" (TOUJOURS cet email exact, JAMAIS varier, JAMAIS inventer d'autre email)

"supplier_siret": "123 456 789 00012" (TOUJOURS exactement cette valeur)

"supplier_vat": "FR12 345678900" (TOUJOURS exactement cette valeur)

Les items doivent être réalistes, variés, et liés à une vente professionnelle (prestations, logiciels, maintenance, licences, services informatiques, conseil…).

RÈGLE CRITIQUE POUR LES DATES :
- TOUTES les dates doivent être en août 2017 uniquement
- Format : "2017-08-XX" où XX est un jour entre 01 et 31
- Exemples : "2017-08-15", "2017-08-03", "2017-08-28"
- JAMAIS de dates en 2025, 2024, etc. UNIQUEMENT août 2017

due_date = date + 30 jours calendaires (format "YYYY-MM-DD"). Exemple : date 2017-08-10 → due_date 2017-09-09.

Les montants doivent être cohérents, et les totaux ht / tva / ttc correctement calculés.

RÈGLE CRITIQUE POUR LA TVA :
- Le champ "tva" doit être un NOMBRE entre 5 et 20 uniquement (ex: 5, 10, 15, 20)
- tva = 5 signifie 5%, tva = 20 signifie 20%
- INTERDIT d'utiliser des valeurs comme 200, 720, 2000, etc.
- Le calcul correct est : total_ttc = total_ht × (1 + tva/100)
- Exemple : si total_ht = 100 et tva = 20, alors total_ttc = 120

N'ajoute aucun commentaire, aucune phrase, rien d'autre que le JSON final.

Le JSON doit être valide et parfaitement structuré, et contenir obligatoirement les champs due_date, supplier_siret, supplier_vat, client_siret et client_vat pour chaque facture.
